# Design System

> **Status**: Active — single source of truth for all visual design
> **Last Updated**: 2026-02-17
> **Scope**: Covers both public marketing pages AND internal /app pages
> **Regenerated by**: `/redesign` skill after visual changes

This document is the **single source of truth** for all visual design decisions. Both public marketing pages and the internal `/app` share the same CSS variables in `globals.css`.

---

## Core Principles

1. **Clean and professional** — No gradients, no decorative fluff, no AI clichés
2. **Functional color** — Color conveys meaning (status, importance, action), not decoration
3. **Consistent spacing** — Use the spacing scale. No arbitrary pixel values.
4. **Card-based layout** — Information is organized in cards with consistent padding
5. **Minimal animation** — Only subtle transitions. No bouncing, pulsing, or attention-seeking animation.

## Color System

### CSS Variables (`globals.css`)

All colors come from CSS variables. **Never use hardcoded Tailwind color classes** for standard UI elements.

| Variable | Value | Usage | Example Classes |
|----------|-------|-------|-----------------|
| `--primary` | `#1a1009` | Brand/accent color | `bg-primary`, `text-primary`, `border-primary` |
| `--primary-foreground` | `#fafafa` | Text on primary backgrounds | `text-primary-foreground` |
| `--foreground` | `#171717` | Main text color | `text-foreground` |
| `--background` | `#ffffff` | Page background | `bg-background` |
| `--muted` | `#fafafa` | Subtle backgrounds | `bg-muted` |
| `--muted-foreground` | `#71717a` | Secondary text | `text-muted-foreground` |
| `--border` | `#e4e4e7` | Borders | `border-border` |
| `--card` | `#ffffff` | Card backgrounds | `bg-card` |
| `--accent` | `var(--muted)` | Hover states | `bg-accent` |
| `--success` | `#008000` | Success states | `text-success`, `bg-success/10` |
| `--warning` | `#f59e0b` | Warning states | `text-warning`, `bg-warning/20` |
| `--danger` | `#C04000` | Error/destructive | `text-danger`, `bg-danger/10` |
| `--info` | `#3b82f6` | Informational | `text-info`, `bg-info/10` |

### Opacity Modifiers

Use opacity modifiers for lighter variations. **Never use separate color shades.**

```typescript
// GOOD
className="bg-primary/10"       // Light primary background
className="bg-danger/20"        // Light danger background
className="text-foreground/80"  // Slightly muted text

// BAD
className="bg-blue-50"          // Use bg-primary/10
className="bg-red-100"          // Use bg-danger/20
className="text-gray-700"       // Use text-foreground/80
```

### When Hardcoded Colors Are Acceptable
- Marketing sections with specific decorative needs
- One-off elements that shouldn't change with theme
- Third-party component styling

### Dark Mode Foundation

`.dark` CSS variable overrides exist in `globals.css` but no toggle UI ships. The `/redesign` skill can generate dark themes using these variables. Add a toggle when ready.

## Typography

### Font Family
- **Primary**: Jost (headings and body)
- **Fallback**: Geist Sans, Geist Mono (for code)

### Heading Hierarchy

| Element | Size | Weight | Class |
|---------|------|--------|-------|
| H1 (hero/page title) | `text-4xl` to `text-5xl` | `font-bold` | Marketing hero: `text-4xl md:text-5xl font-bold tracking-tight` |

## Spacing

### Base Unit
4px (Tailwind default). All spacing uses the Tailwind scale.

### Common Spacing

| Context | Value | Tailwind |
|---------|-------|----------|
| Tight inline | 4px | `gap-1` |
| Icon-to-text | 8px | `gap-2` |
| Between related items | 12px | `gap-3` |
| Between sections | 16-24px | `gap-4` to `gap-6` |
| Page section spacing | 48-96px | `py-12` to `py-24` |
| Large vertical gap | 64px | `gap-16` |

### Container Patterns

| Context | Width | Classes |
|---------|-------|---------|
| App content | Full width with padding | `px-4 md:px-6` |
| Marketing narrow | 768px | `max-w-3xl mx-auto px-4` |
| Marketing standard | 1200px | `max-w-7xl mx-auto px-4 md:px-8` |
| Marketing wide | 1400px | `max-w-[1400px] mx-auto px-4 md:px-8` |

### Card Padding
- Standard card: `p-6`
- Compact card: `p-4`
- Card with header: header `px-6 py-4`, content `p-6 pt-0`

## Components

### Buttons

Always use `Button` from `@/components/ui/button`. Available variants:

| Variant | Appearance | Usage |
|---------|-----------|-------|
| `default` | Primary filled | Main actions |
| `outline` | Border only | Secondary actions |
| `ghost` | No border | Tertiary actions |
| `danger` | Soft red (`bg-danger/10 text-danger`) | Destructive actions |
| `link` | Text only with underline | Navigation |

**Destructive actions**: Always use `variant="danger"` (soft red). NEVER use hard red (`bg-red-600`). ALWAYS show `AlertDialog` confirmation first.

**Hero CTA** (marketing pages):
```typescript
<Button size="lg" className="gap-2">
  Get Started <ArrowRight className="h-4 w-4" />
</Button>
```

### Cards

```typescript
<Card>
  <CardHeader>
    <CardTitle>Title</CardTitle>
    <CardDescription>Description</CardDescription>
  </CardHeader>
  <CardContent>
    {/* Content */}
  </CardContent>
</Card>
```

### Badges

Status badges use semantic colors:
```typescript
<span className="inline-flex items-center px-2 py-0.5 text-xs font-medium rounded-full bg-success/10 text-success border border-success/30">
  Active
</span>
```

### Info Tooltips

Non-obvious features should have an `Info` icon with tooltip:
```typescript
<TooltipProvider>
  <Tooltip>
    <TooltipTrigger asChild>
      <button className="text-muted-foreground hover:text-foreground">
        <Info className="h-4 w-4" />
      </button>
    </TooltipTrigger>
    <TooltipContent className="max-w-xs">
      <p className="text-sm">Explanation of why this matters</p>
    </TooltipContent>
  </Tooltip>
</TooltipProvider>
```

### Tabs

Use shadcn Tabs for any toggle/filter UI:
```typescript
<Tabs value={selected} onValueChange={setSelected}>
  <TabsList>
    <TabsTrigger value="all">All</TabsTrigger>
    <TabsTrigger value="active">Active</TabsTrigger>
  </TabsList>
</Tabs>
```

## Anti-Patterns (NEVER Do These)

1. **No gradients** — No `bg-gradient-to-r`, no gradient text, no gradient backgrounds
2. **No purple-blue combinations** — No `bg-purple-600`, `text-blue-500` paired together
3. **No Sparkles icon** — Banned. Use `Wand2`, `RefreshCw`, `Zap`, `Star` instead.
4. **No hardcoded color classes** for standard UI — Use CSS variables
5. **No inline error text** that pushes content down — Use tooltips on icons
6. **No hard red buttons** (`bg-red-600`) — Use `variant="danger"` (soft red)
7. **No custom button groups** — Use shadcn Tabs component for toggles
8. **No destructive actions without confirmation** — Always show AlertDialog first
9. **No arbitrary pixel values** — Use Tailwind spacing scale
10. **No decorative animations** — Only functional transitions (`transition-colors`, `transition-opacity`)
